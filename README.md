## 🎯 Objective

> **Build, evaluate, and compare supervised regression models to accurately predict house prices — while mastering gradient descent optimization and diagnosing model behavior through bias-variance analysis.**

This project is designed to achieve the following goals:

- ✅ Understand and apply **Supervised Learning** principles in a real-world context
- ✅ Implement **Simple**, **Multiple**, and **Polynomial** Linear Regression models
- ✅ Build all three **Gradient Descent** variants from scratch using pure NumPy
- ✅ Diagnose **overfitting**, **underfitting**, and the **bias-variance trade-off**
- ✅ Evaluate models using industry-standard metrics: MAE, MSE, RMSE, R², Adjusted R²
- ✅ Deliver actionable insights applicable to a real-world real estate analytics firm

---

## 🏢 Problem Statement

**Role:** Junior Data Scientist at a Real Estate Analytics Firm

A real estate company needs a reliable predictive system to estimate property prices based on measurable features. Manual valuation is slow, inconsistent, and difficult to scale. The company seeks a data-driven machine learning solution to:

- 🔎 Identify which property features have the highest impact on price
- 🤖 Build and compare regression models for price prediction
- ⚡ Optimize model learning using gradient descent techniques
- 📊 Select the best-performing model for deployment

**Core Question:** *Which regression model most accurately predicts house prices while generalizing well to unseen data?*

## 📚 Theoretical Concepts Covered

| # | Concept | Description |
|---|---|---|
| 1️⃣ | **Supervised Learning** | Learning from labeled (input → output) data to predict outcomes on unseen samples |
| 2️⃣ | **Regression vs Classification** | Regression predicts continuous values; Classification predicts discrete categories |
| 3️⃣ | **Simple Linear Regression** | One input feature → one output using equation Y = mX + c |
| 4️⃣ | **Multiple Linear Regression** | Multiple input features → one continuous output |
| 5️⃣ | **Polynomial Regression** | Extends linear regression to model non-linear relationships via degree-n features |
| 6️⃣ | **Assumptions of Linear Regression** | Linearity, Independence, Homoscedasticity, Normality, No Multicollinearity |
| 7️⃣ | **Gradient Descent** | Iterative optimization algorithm that minimizes the cost function by updating weights |
| 8️⃣ | **Bias-Variance Trade-Off** | Balance between model simplicity (bias) and sensitivity to training data (variance) |
| 9️⃣ | **Overfitting** | Model memorizes training data; performs poorly on new data — high variance |
| 🔟 | **Underfitting** | Model is too simple to capture data patterns — high bias |

## 📐 Model Evaluation Metrics

| Metric | Full Form | What It Tells You |
|---|---|---|
| 📏 **MAE** | Mean Absolute Error | Average magnitude of errors — easy to interpret, same unit as price |
| 📐 **MSE** | Mean Squared Error | Penalizes large errors more heavily — useful for catching outlier mistakes |
| 📊 **RMSE** | Root Mean Squared Error | Square root of MSE — interpretable in the same unit as the target (INR) |
| 🎯 **R² Score** | Coefficient of Determination | % of price variance explained by the model (1.0 = perfect) |
| 📈 **Adjusted R²** | Adjusted Coefficient of Determination | R² corrected for number of features — prevents inflation from adding weak features |

> 💡 **Lower MAE / MSE / RMSE = Better. Higher R² and Adjusted R² = Better.**

## ⚙️ Gradient Descent Optimization

All three gradient descent variants were **implemented from scratch using pure NumPy** — no library shortcuts.

### 🔵 Batch Gradient Descent
- Updates weights using the **entire dataset** per iteration
- Convergence is **smooth and stable**
- Drawback: **Slow** — computationally expensive for large datasets

### 🟢 Stochastic Gradient Descent (SGD)
- Updates weights using **one random sample** per iteration
- **Fastest** update cycle but **noisy convergence**
- Good for online learning; unpredictable path to minimum

### 🟡 Mini-Batch Gradient Descent
- Updates weights using a **small batch of samples** (batch_size = 32)
- **Balances speed and stability** — best of both worlds
- Industry standard for real-world deep learning and ML

### ⚡ Convergence Comparison

| Method | Update Per Step | Convergence | Speed | Best Use Case |
|---|---|---|---|---|
| ⚡ Batch GD | Full dataset | 🟢 Smooth & Stable | 🐢 Slowest | Small datasets |
| 🎲 SGD | 1 sample | 🔴 Noisy & Erratic | 🚀 Fastest | Online / streaming data |
| ⚖️ Mini-Batch GD | 32 samples | 🟡 Mostly Stable | 🏎️ Balanced | ✅ General purpose (recommended) |

> 🏆 **Mini-Batch GD is the winner** — converges faster than Batch GD and is more stable than SGD.

## 📉 Bias-Variance & Model Diagnostics

### 🔬 Analysis Per Model

| Model | Bias Level | Variance Level | Complexity | Diagnosis |
|---|---|---|---|---|
| Simple Linear Regression | 🔴 High | 🟢 Low | ⬛ Low | **Underfitting** — one feature is not enough |
| Multiple Linear Regression | 🟢 Low | 🟡 Moderate | ⬛⬛⬛ Medium | ✅ **Ideal** — good fit, generalizes well |
| Polynomial Regression (deg 2) | 🟡 Medium | 🟡 Medium–High | ⬛⬛⬛⬛ High | **Watch degree** — can overfit at deg 3+ |

### 📊 How Model Complexity Affects Error

```
 Error
   ↑
   │  ╲                                ╱
   │   ╲    Underfitting         Overfitting
   │    ╲                         ╱
   │     ╲_______________________╱   ← Optimal Zone (Multiple LR)
   │
   └──────────────────────────────────→  Model Complexity
        Simple LR      Multiple LR      High-Deg Polynomial
```

- 🔴 **Too Simple** → High training error + high testing error = Underfitting
- ✅ **Just Right** → Low training error + low testing error = Good Generalization
- 🟠 **Too Complex** → Very low training error + high testing error = Overfitting

> 🏆 **Best Bias-Variance Balance: Multiple Linear Regression**

## ⚠️ Challenges Faced

These are the real problems encountered and resolved during the project:

### 🔴 Challenge 1 — Underfitting with Simple Linear Regression
> **Problem:** Using only `area_sqft` produced an R² of ~0.56 — the model missed critical pricing signals.
> **Solution:** Extended to Multiple Linear Regression using all 5 features, boosting R² to ~0.91.

### 🔴 Challenge 2 — Noisy Convergence in SGD
> **Problem:** Stochastic Gradient Descent produced erratic cost curves — hard to confirm convergence.
> **Solution:** Compared all three GD methods visually; confirmed Mini-Batch as the most reliable approach.

### 🔴 Challenge 3 — Overfitting Risk in Polynomial Regression
> **Problem:** At higher polynomial degrees (3+), the model began memorizing training data and generalization dropped.
> **Solution:** Restricted polynomial degree to 2 and monitored training vs testing error gap carefully.

### 🔴 Challenge 4 — Choosing the Right Evaluation Metric
> **Problem:** R² alone can be misleading — adding irrelevant features can inflate it artificially.
> **Solution:** Used **Adjusted R²** alongside MAE, MSE, and RMSE for a comprehensive, honest evaluation.

### 🔴 Challenge 5 — Implementing Gradient Descent from Scratch
> **Problem:** Getting weight and bias updates mathematically correct without library shortcuts required careful derivation of the MSE cost function gradient.
> **Solution:** Derived update rules manually and validated outputs against scikit-learn's results.
